## Getting started

<Steps>
  <Step title="Install Cookiecutter">
    Install Cookiecutter using `uv` (or pip):

    ```bash
    uv tool install "cookiecutter>=1.7.0"
    ```
  </Step>
  <Step title="Generate your project">
    Run Cookiecutter against this repository and answer the prompts:

    ```bash
    uvx cookiecutter https://github.com/cookiecutter/cookiecutter-django
    ```
  </Step>
  <Step title="Set up your environment">
    Enter your project directory, install dependencies, and run migrations:

    ```bash
    cd <your_project_slug>
    uv sync
    uv run python manage.py migrate
    uv run python manage.py runserver
    ```
  </Step>
  <Step title="Explore and customize">
    Your project is running at [http://localhost:8000](http://localhost:8000). Check out the [project generation options](/project-generation-options) to understand every choice you made, or dive into the [local development guide](/local-development/developing-locally) for a full walkthrough.
  </Step>
</Steps>

## Optional integrations

Enable any of these features during project generation:

| Integration | Option | Description |
|---|---|---|
| Celery | `use_celery=y` | Background task processing with Redis broker |
| Sentry | `use_sentry=y` | Error tracking and performance monitoring |
| Mailpit | `use_mailpit=y` | Local email testing without a real mail server |
| Whitenoise | `use_whitenoise=y` | Serve static files directly from Django |
| Heroku | `use_heroku=y` | `Procfile` and Heroku-ready configuration |
| Async/WebSockets | `use_async=y` | ASGI setup with Uvicorn + Gunicorn |

<Note>
  All optional integrations can only be enabled at project generation time. Plan your choices before running `cookiecutter`.
</Note>
