---
title: "Quickstart: generate a Django project in minutes"
description: "Go from zero to a running Django project in minutes — generate a fully configured, production-ready codebase with a single command."
---

Cookiecutter Django scaffolds a complete Django project from a template by asking you a series of questions. The result is a working codebase with your chosen database version, cloud provider, email service, and other integrations already wired up.

## Prerequisites

Before you begin, make sure you have the following installed:

- **uv** (Python package manager) — [install uv](https://docs.astral.sh/uv/getting-started/installation/)
- **Python 3.14** — installed automatically by uv if needed
- **PostgreSQL** — a running instance for running migrations locally

## Generate your project

<Steps>
  <Step title="Install Cookiecutter">
    Use `uv tool install` to make the `cookiecutter` command available globally:

    ```bash
    uv tool install "cookiecutter>=1.7.0"
    ```
  </Step>

  <Step title="Run Cookiecutter against this repo">
    Point Cookiecutter at the cookiecutter-django GitHub repository:

    ```bash
    uvx cookiecutter https://github.com/cookiecutter/cookiecutter-django
    ```

    You will be prompted to answer a series of questions that configure your project. See the [project generation options reference](/project-generation-options) for a full description of every prompt.
  </Step>

  <Step title="Enter the generated directory">
    The generated project directory is named after the `project_slug` you provided. Enter it and look around:

    ```bash
    cd reddit/
    ls
    ```
  </Step>

  <Step title="Run database migrations">
    Apply the initial Django migrations against your local PostgreSQL database:

    ```bash
    uv run python manage.py migrate
    ```
  </Step>

  <Step title="Start the development server">
    Start the Django development server:

    ```bash
    uv run python manage.py runserver 0.0.0.0:8000
    ```

    Your project is now running at `http://localhost:8000`.
  </Step>
</Steps>

## Review the generated README

The generated project includes its own `README` with instructions specific to your chosen configuration — including environment variable setup, local development commands, and deployment steps. Read it carefully before proceeding.
