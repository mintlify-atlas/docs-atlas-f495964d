title: "Environment variables and settings reference"
description: "Configure your Cookiecutter Django project using environment variables following the 12-Factor App methodology, powered by django-environ."
---

Cookiecutter Django follows the [12-Factor App](https://12factor.net/config) methodology for configuration, using [django-environ](https://django-environ.readthedocs.io/) to read settings from environment variables. This approach keeps secrets out of your codebase and makes it easy to configure different environments without changing code. Note that this setup **does not work with Apache/mod_wsgi** — environment variables are only picked up when running under Gunicorn/Nginx or uWSGI/Nginx.

<Tip>
  You can store environment variables in a `.env` file at the root of your project. Set `DJANGO_READ_DOT_ENV_FILE=True` to have django-environ load it automatically. OS-level environment variables always take precedence over values in `.env`.
</Tip>

## Other environment settings

These settings control application-level behaviour beyond what Django provides natively.

**`DJANGO_ACCOUNT_ALLOW_REGISTRATION`** (default: `True`)

Allows you to enable or disable user registration through [django-allauth](https://docs.allauth.org/) without disabling other features like authentication and account management. Maps to the `ACCOUNT_ALLOW_REGISTRATION` Django setting.

**`DJANGO_ADMIN_FORCE_ALLAUTH`** (default: `False`)

Forces the Django admin sign-in process to go through the django-allauth workflow, ensuring consistent authentication behaviour across your project.
