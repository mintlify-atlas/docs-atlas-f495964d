title: Deploy Cookiecutter Django on Heroku
description: Deploy your Cookiecutter Django project to Heroku using the Heroku CLI — provision add-ons, set config vars, and push your code to go live.
---

Heroku is a managed platform-as-a-service that lets you deploy Django applications without managing servers directly. Cookiecutter Django includes a `Procfile` and production settings that work with Heroku's build system out of the box. The steps below walk through a first-time deployment using the Heroku CLI.

## Deployment script

Run these commands in order to provision and deploy your application:

```bash
heroku create --buildpack heroku/python

# Note: this is not a free plan
heroku addons:create heroku-postgresql:essential-0

# On Windows use double quotes for the time zone, e.g.
# heroku pg:backups schedule --at "02:00 America/Los_Angeles" DATABASE_URL
heroku pg:backups schedule --at '02:00 America/Los_Angeles' DATABASE_URL
heroku pg:promote DATABASE_URL

heroku addons:create heroku-redis:mini

# Assuming you chose Mailgun as mail service (see below for others)
heroku addons:create mailgun:starter

heroku config:set DJANGO_DEBUG=False
heroku config:set DJANGO_SETTINGS_MODULE=config.settings.production
heroku config:set DJANGO_SECRET_KEY="$(openssl rand -base64 64)"

# Generating a 32 character-long random string without any of the visually similar characters "IOl01":
heroku config:set DJANGO_ADMIN_URL="$(openssl rand -base64 4096 | tr -dc 'A-HJ-NP-Za-km-z2-9' | head -c 32)/"

# Set this to your Heroku app url, e.g. 'bionic-beaver-28392.herokuapp.com'
heroku config:set DJANGO_ALLOWED_HOSTS=

# Assign with AWS_ACCESS_KEY_ID
heroku config:set DJANGO_AWS_ACCESS_KEY_ID=

# Assign with AWS_SECRET_ACCESS_KEY
heroku config:set DJANGO_AWS_SECRET_ACCESS_KEY=

# Assign with AWS_STORAGE_BUCKET_NAME
heroku config:set DJANGO_AWS_STORAGE_BUCKET_NAME=

git push heroku main

heroku run python manage.py createsuperuser

heroku run python manage.py check --deploy

heroku open
```

## Notes

### Email service

The script above assumes you chose Mailgun as your email service. If you want to use a different provider, refer to the [django-anymail documentation](https://anymail.readthedocs.io) for the environment variables each provider requires. Heroku also offers other [email add-ons](https://elements.heroku.com/addons#email-sms) such as SendGrid, which can be configured with a similar one-line command.

<Warning>
  If your email server is not configured properly (Mailgun by default), attempting to send an email will cause an Internal Server Error.

  By default, `django-allauth` is set up to require email verification, which means it sends a verification email when an unverified user tries to log in or when someone signs up.

  This can happen immediately after setting up a Mailgun account, which runs in a sandbox subdomain by default. Either add your email to the list of authorized recipients or verify your domain.
</Warning>

### Heroku and Docker

Although Heroku offers some [Docker support](https://devcenter.heroku.com/articles/build-docker-images-heroku-yml), it is not supported by Cookiecutter Django. Refer to the Heroku documentation if you want to explore that path independently.

## Optional actions

### Celery

Celery requires a few extra environment variables to work correctly. The worker process is defined in the `Procfile` but is turned off by default. To enable it, scale the worker dyno to one instance:

```bash
heroku ps:scale worker=1
```

### Sentry

If you opted for Sentry error tracking, you can install it as a Heroku add-on:

```bash
heroku addons:create sentry:f1
```

Or, if you already have a Sentry account, set the DSN directly:

```bash
heroku config:set SENTRY_DSN=https://xxxx@sentry.io/12345
```

### Gulp or Webpack

If you chose Gulp or Webpack as your frontend pipeline, you need to configure [multiple buildpacks](https://devcenter.heroku.com/articles/managing-buildpacks) — one for Python and one for Node.js:

```bash
heroku buildpacks:add --index 1 heroku/nodejs
```

During deployment, Heroku will run `npm install` and then `npm build`, which handles SASS compilation and JavaScript bundling. If you run into issues, refer to the Heroku buildpack documentation.
